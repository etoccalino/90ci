# Candidate rules

- IF building/serving the web front-end in this environment, THEN do NOT use `pnpm` — the repo pins pnpm 11 which requires Node ≥ 22.13, but the environment has Node v20.20.2, so pnpm crashes on the missing `node:sqlite` builtin. Drive the tools directly instead: build the WASM with `sh scripts/build-wasm.sh`, build the front-end with `node web/node_modules/vite/bin/vite.js build` (from `web/`), and typecheck with `node web/node_modules/typescript/bin/tsc --noEmit -p tsconfig.json`. BECAUSE the pnpm scripts (`build`, `dev`) are unusable here and the underlying Rust/Vite toolchain works fine when invoked directly.
- IF smoke-testing the WASM compute path without a browser, THEN import `web/src/wasm/ninety_ci_wasm.js` from a Node ESM script and call the default `init()` with the wasm bytes (`init(readFileSync(...ninety_ci_wasm_bg.wasm))`), then call `simulate(...)`. BECAUSE no headless browser (chromium/playwright/puppeteer) is installed, but the `--target web` glue accepts raw bytes and runs under Node.
